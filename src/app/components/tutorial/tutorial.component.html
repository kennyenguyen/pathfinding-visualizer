Welcome to Pathfinding Visualizer

This quick tutorial will walk you through all of the features of this app.

If you want to dive right in, feel free to click the "Skip Tutorial" button below. Otherwise, click "Next" to continue!

What is a pathfinding algorithm?

Pathfinding algorithms seek to find the shortest path between two points. This app visualizes various pathfinding algorithms in action.

All of the algorithms in this app are adapted for a 2D grid, where movements from one node to another have a "cost" of 1.

Picking an algorithm

Choose an algorithm from the "Algorithms" drop-down menu.

Note that some algorithms are weighted, while others are unweighted. Unweighted algorithms do not take turns or weight nodes into account, whereas weighted ones do. Additionally, not all algorithms can guarantee the shortest path.

What is a pathfinding algorithm?

Not all algorithms are created equal.
Dijkstra's Algorithm (weighted): the father of all pathfinding algorithms; guarantees the shortest path.
A* Search (weighted): arguably the best pathfinding algorithm; uses heuristics to guarantee the shortest path much faster than Dijkstra's.
Breadth-First Search (BFS) (unweighted): a great algorithm; guarantees the shortest path.
Depth-First Search (DFS) (unweighted): a very bad algorithm for pathfinding; does not guarantee the shortest path.

Adding walls and weights

Click on the grid to add a wall. Click "Add Weight" and then click on the grid to add a weight.

Walls are impenetrable, meaning that paths cannot cross through them. Weights, however, are not impassable. They are simply more "costly" to move through. In this app, moving through a weight node has a "cost" of 10.

Dragging nodes

Click and drag the start and end nodes to move them.

Note that you can drag nodes even after an algorithm has finished running. This will allow you to instantly see different paths.

Visualizing and more

Use the navbar buttons to visualize algorithms, and do other stuff!

You can clear the current path, any walls or weights, or even the entire board, all from the navbar.
